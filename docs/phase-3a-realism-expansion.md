# Phase 3A-REALISM-EXPANSION: Controlled Human-Quality Content Scaling

## Purpose

This phase resumes content growth after the editorial realism overhaul, but only through small reviewed batches. The aim is not to restore bulk scale quickly. The aim is to grow the IELTS Academic Reading library while preserving human-writing realism, plausible distractors, real evidence excerpts, answer-balance discipline and taxonomy consistency.

## Controlled Expansion Plan

Batch A was the first validation batch after the realism reset:

- Add 3 mini tests.
- Add 8 drill-native focused practice sets.
- Preserve all 3 flagship benchmark tests.
- Publish only content marked as reviewed and validated.
- Run structural, metadata, evidence, answer-balance and repetition checks before treating the batch as usable.

Controlled Batch B has now continued the same process:

- Add 3 mini tests.
- Add 8 drill-native focused practice sets.
- Prioritize under-covered question types and skills.
- Publish only after the same validation checks pass with no warnings.

Future controlled batches:

- Batch C: 2 Band 8-9 challenge tests and 5 advanced challenge drills.
- Batch D: targeted repair and expansion for Matching Headings, Matching Information, Matching Features, Flow-chart Completion and Diagram Label Completion.

The content validator checks for repeated question-type sequences across the published test set.

## Distractor Quality Report

New MCQ and matching options use more plausible wrong-answer logic:

- Partial truth.
- Scope distortion.
- Cause-effect confusion.
- Chronology confusion.
- Similar keyword attraction.
- Unsupported inference.
- Wrong emphasis.

Distractors should sound academically possible under time pressure while remaining contradicted or unsupported by the passage.

## Evidence Integrity Report

Each new question includes:

- A paragraph reference.
- A real excerpt or close excerpt from the passage.
- Explanation of the meaning relationship.
- Why-correct and why-wrong notes.
- Strategy guidance tied to the trap or skill.

The validator warns against generic evidence summaries such as `Paragraph C states this directly`.

## Answer-Position Balance Report

Batch A specifically corrected a matching-sentence answer-position concentration issue caught by validation. Batch B preserved that balance after adding eight drills and three tests. The published set now passes answer-position checks with no warnings.

Validation looks for:

- Repeated MCQ correct positions.
- Repeated option dominance.
- Repeated answer words.
- Repeated paragraph targets.
- Repeated heading or sentence-ending positions.

## Human-Realism QA Report

Batch A and Batch B content was written to avoid the old generator pattern of evenly balanced paragraphs, generic research language and predictable endings. New passages use discipline-specific details, varied sentence rhythm and less mechanical transitions.

Remaining human-realism risks:

- The library is still small enough that repeated editorial habits can become visible quickly.
- Strategy lessons still need a separate prose rewrite.
- Further batches should include more humanities and social-science voices, even though Batch B added archaeology and linguistics coverage.

## Psychometric QA Report

The controlled batches improve assessment credibility through:

- More varied cognitive demand.
- Better distribution of question types.
- More plausible distractors.
- Real evidence excerpts.
- Validation against answer-position bias.
- Clear distinction between direct, paraphrased, inferred and contrast-based evidence.
- Coverage for every standard question type, reading skill and major trap type.

Remaining psychometric risks:

- Diagram-label completion is still a text-based approximation.
- Matching Headings and Matching Features have better coverage but still need more batch-specific variety before being considered deeply covered.
- Band 8-9 content needs more external human review before long-term scale.

## Validation Guards

Run before publishing:

```bash
npm run validate:content
npx tsc --noEmit
npm run lint
npm run build
```

Current guards include:

- Taxonomy consistency.
- Metadata consistency.
- Evidence references.
- Academic-only scope.
- Drill lesson links.
- Recommendation relationships.
- Answer-position concentration.
- Repeated question-type sequences.
- Repeated paragraph openings.
- Generic evidence wording.
- Coverage reporting for held-back drill areas.

## Remaining Weaknesses

- The platform has not yet reached the target of 15 high-quality mini tests.
- The platform has not yet reached the target of 40 to 60 high-quality drills.
- Every standard question type now has at least one drill, but many have only one and need deeper variety.
- Strategy lessons need editorial rewriting to match the improved item quality.
- Visual support is still needed for authentic Diagram Label Completion.
- Broader topic diversity is needed before the library feels mature.
